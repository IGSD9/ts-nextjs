# **チーム・コンディション・チェッカー**

# **要件定義書**

## **1\. プロジェクト概要**

本システムは、チームメンバーの心身の状態、および業務の透明性（「プロジェクトの霧」）を最小限の負荷（3秒）で可視化することを目的とする。収集したデータに基づき、燃え尽き症候群の防止、心理的安全性の向上、および業務停滞の早期検知を支援する。

## 

## **2\. コア・コンセプト**

* **「3秒」のUX**: 思考を挟まず、直感的な絵文字タップだけで完了する操作性。  
* **摩擦ゼロのログイン**: パスワード不要の「マジックリンク」と、初回チェックイン時の「オートプロビジョニング（自動ユーザー作成）」。  
* **心理的安全性**: 匿名でのポジティブ・フィードバック（チーム・メッセージ）による関係性の質向上。

## 

## **3\. 機能要件**

### **3.1 認証・ユーザー管理**

* **マジックリンク認証**: メールアドレス入力のみでログインリンクを送信。パスワード管理のストレスを排除。  
* **ユーザー自動生成**: 初回ログイン・チェックイン時にDB上にユーザー情報を自動作成。  
* **プロフィール管理**: 設定画面から表示名の変更が可能。

### 

### **3.2 超高速チェックイン（入力機能）**

以下の6カテゴリを絵文字で選択。各項目には直感的な回答を促す補助説明を付与。

| カテゴリ | ラベル | 補助説明（問いかけ） | 選択肢（絵文字） |
| :---- | :---- | :---- | :---- |
| **身体** | 睡眠 | ぐっすり眠れて、疲れは取れていますか？ | 💤 / 🥱 |
| **身体** | 体の重さ | 肩こりや目の疲れ、痛みはないですか？ | 🏃 / 🤕 |
| **精神** | 今の気分 | 漠然とした不安やイライラはないですか？ | 😊 / 😐 / 🌀 |
| **状況** | プロジェクトの霧 | 次に何をすべきか、進め方は見えていますか？ | ☀️ / ☁️ / 🌫️ |
| **意欲** | バッテリー | 仕事に取り組む気力は残っていますか？ | 🔋 / 🪫 |
| **繋がり** | 繋がり | 今日、誰かと雑談や相談をしましたか？ | 🗣️ / 😶 |

* **動的要素**: 水曜日のみ追加のライトな質問（例：最近笑った？）を表示。

### **3.3 フィードバック・演出（飽き防止）**

* **Message from Team**: チェックイン完了直後に、匿名化されたポジティブな感謝メッセージをランダムに1つ表示。  
* **ピザ・メーター**: チーム全体の入力率や元気度に応じてゲージが溜まり、満タンで「ピザ・バッジ（親睦のきっかけ）」を発動。

### **3.4 ダッシュボード（可視化）**

* **個人用（My Dashboard）**:  
  * 過去7日間のコンディション推移を「お天気図」形式で表示。  
  * 入力傾向に基づく「AIアドバイス（セルフケア推奨）」の表示。  
* **管理者用（Admin Console）**:  
  * **閲覧権限**: 管理者ロールのみアクセス可能。  
  * **FOGアラート**: 5日間連続で「濃霧（🌫️）」を選択したメンバーの兆候を検知し警告を表示。  
  * **チーム統計**: 全体の元気度、参加率をサマリー表示。  
  * **デザイン**: 一般画面と統一感のあるライトテーマを採用。

## **4\. 技術要件（テクニカル・スタック）**

| レイヤー | 選定技術 | 備考 |
| :---- | :---- | :---- |
| **Framework** | **Next.js (App Router)** | 高速表示とServer ActionsによるDB操作。 |
| **Database** | **Supabase (PostgreSQL)** | 認証、DB、リアルタイム機能を統合。 |
| **ORM** | **Prisma** | TypeScriptとの親和性が高い型安全な開発。 |
| **Deployment** | **Vercel** | 東京リージョン（HND1）設定による低遅延。 |
| **UI/UX** | **Tailwind CSS / Lucide React** | モダンで清潔感のある、スマホ最適化UI。 |
| **App化** | **PWA** | ホーム画面追加による「アプリ」としての起動。 |

## **5\. データモデル (Prisma Schema)**

model User {  
  id        String   @id @default(cuid())  
  email     String   @unique  
  name      String   // 初期値はEmailのアカウント名。後で変更可能。  
  role      String   @default("user") // "user" or "admin"  
  reports   DailyReport\[\]  
}

model DailyReport {  
  id           String   @id @default(cuid())  
  userId       String  
  user         User     @relation(fields: \[userId\], references: \[id\])  
  sleep        Int      // 1-2  
  body         Int      // 1-2  
  mood         Int      // 1-3  
  fog          Int      // 1-3  
  energy       Int      // 1-2  
  connection   Int      // 1-2  
  createdAt    DateTime @default(now())  
}

model PositiveMessage {  
  id        String   @id @default(cuid())  
  content   String   // 例: "資料の図解が分かりやすかったです！"  
}

## **6\. 非機能要件**

* **パフォーマンス**: 起動から入力完了まで10秒以内（うち、入力作業自体は3秒）。  
* **セキュリティ**:  
  * データベースURLおよび認証キーの環境変数による秘匿。  
  * 管理者機能への厳格なアクセス制御。
